CARLA Nav2

ROS2-based autonomous navigation stack. CARLA + Humble + Navigation2.

Overview

A complete navigation system built on industry-standard components. CARLA for simulation, ROS2 for middleware, Navigation2 for planning and control. Designed to transfer directly from simulation to real hardware.

Repository: github.com/arassal/carla-nav2-avl

Capabilities

Lane Following

Pure pursuit with adaptive lookahead. Tracks lane centerlines at variable speeds. Handles multi-lane roads and curves.

Obstacle Detection

Lidar-based 2D costmap. Real-time obstacle detection with emergency braking. Safety-critical systems.

Traffic Lights

Automatic detection and enforcement. Stops on red, resumes on green. Full rule compliance.

Multi-Sensor Fusion

3 RGB cameras, 64-ray lidar, IMU, GNSS. Time-synchronized sensor data pipeline.

Real-Time Control

Deterministic control loop. Published on standard ROS2 topics. Ready for hardware deployment.

Path Planning

Navigation2 integration. Navfn planner. Behavior trees for mission logic.

Stack

Simulation

CARLA 0.10.0 (UE5). Native ROS2 sensor publishing. Python API for vehicle control.

Middleware

ROS2 Humble. DDS on domain 0. Standard message types (sensor_msgs, nav_msgs, geometry_msgs).

Navigation

Navigation2 stack. Costmap 2D. Pure pursuit controller. Lifecycle manager.

Development

Python 3.10/3.11. Ubuntu 22.04. Git for version control.

Full setup guide: README on GitHub

Architecture

Four-layer stack: Hardware → ROS2 → Navigation2 → Application

Layer 1 - Hardware: GPU, cameras, lidar, CPU. Ubuntu 22.04.
Layer 2 - ROS2: DDS middleware. TF2 transforms. Standard message types.
Layer 3 - Navigation2: Costmap 2D. Path planner. Controller. Behavior trees.
Layer 4 - Application: CARLA bridge. Custom nodes. Visualization.
